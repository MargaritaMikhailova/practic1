## Курс по автоматизации 


### GIT 


#### Что такое Git


**Итерация** - повторение, начало какого-то действия или шага. 


**Version Control System (VCS)** - система контроля версий


**Source Control Management(SCM)** - система управления исходным кодом 


Одна из ключевых особенностей современных систем контроля версий - поддержка паралельной работы нескольких пользователей в том числе под одним файлом, именно поэтому *VCS* популярны у IT-команд.


**GIT** - **Global Information Tracker** - глобальный информационный трекер. 


Сиситема контроля версий или VCD(SCM) - программа позволяющая контролировать изменения в проекте 


GIT - один из примеров системы контроля версий, он позволяет хранить, изменять и анализировать историю проекта.


GIT - незаменим в команде инструмент, ведь он помогает объединить результаты работы нескольких человек.


macOS/Linux - ОС на системе Unix, Windows - отдельная ОС 


Для работы с командной строкой на Windows нужен Git Bash 


### Командная строка 


``` pwd ``` - показать рабочую папку


``` cd ``` - сменить директорию 


``` ~ ``` - домашняя директория 


Консольный и графический интерфейсы два  способоа вхаимодействия с программами 


* Консоль, терминал, или командная строка - это программы, которые считывает команду пользователя и выполняет ее 


* Чтобы вывести текущую рабочую директорию, можно испольщовать команду ``` pwd ```


* У большинства пользователей компьютера есть доступ к домашней директории, чтобы к ней перейти, используют команду ``` cd ~ ```


* ``` ls ``` - вывести содержимое директории


* Если в названии есть пробелы, при вводе нужно использовать кавычки  ``` $ cd "Фотографии с дня рождения" ```


Чтоббы вернуться в родительскую директорию - то есть на уровень выше - вместо названия нужно писать две точки ``` .. ```


``` pwd 


/project/github *# сейчас мы здесь* 


$ .. # переходим на уровень выше ```


``` . ``` - точка, чтобы обратиться к текущей директории 


``` -a ``` - расширенный список 


``` touch ``` - создание файла 


``` mkdir ``` - создание директории 


``` -p ``` - создать целую структуру директории 


``` $ mkdir -p dir1/dir-inside/dir-deep-inside  * # создам папку dir-deep-inside в папке dir-inside, которая находится в папке dir ```


Также можно использовать обе команды вместе с симоволом домашней директориии (~) или родительской директории (..). Например команда *mkdir ~/my-git-project* создаст папку внутри **my-git-project** внутри домашней директории. А команда *touch ../../file.txt на две папки выше по иерархии. Допусти, если вы находитесь в директории * project/git/hello* команда *touch ../../file.txt* создаст файл по такому пути project/file.txt


``` cp ``` - копирование файлов 

``` $ cp *что_копируем и куда_копируем


$ cp index.html src/  *# скопировали index.html в нашу папку src ```


``` mr ``` - перемещение файлов и папок 


* команда *touch* создает файл, а команда *mkdir* создает директорию(папку)


* с помощью флага *-p* можно создать целую структуру директории одной командой ``` mkdir -p ```


* для копирования файлов используют команду *cp*, для перемещения *mv*
